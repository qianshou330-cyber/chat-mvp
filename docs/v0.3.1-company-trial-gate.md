# v0.3.1 公司试用闸门

## 当前目标

`v0.3.1-beta` 是 20-30 人公司内部试用前的收口版本。它不新增大功能，只补试用说明入口、生产 smoke test、测试数据清理和安全检查。

## 试用前检查

- 生产站点：`https://chat-mvp-tau.vercel.app`
- 基线标签：`v0.3-beta`
- 目标标签：`v0.3.1-beta`
- 必须通过：登录、工作区管理、成员添加/移除、好友申请、单聊、群聊、附件、搜索、通知、登录设备。
- 必须确认：非成员不能读取工作区群聊、附件或 push subscription。
- 必须确认：Vercel、GitHub、前端代码不包含 Supabase secret、service role、VAPID private key 或 webhook secret。

## 试用执行方式

1. 先邀请 5 个核心用户试用 1 天。
2. 5 人先导期间只修 P0/P1，不夹带大功能。
3. 先导无 P0/P1 后扩到 20-30 人。
4. 创建 1 个正式试用工作区，由 owner/admin 通过已注册邮箱添加成员。
5. 所有反馈进入 GitHub Issues，并按 P0/P1/P2/P3 分类。

## P0/P1 判断

- P0：无法登录、消息无法发送、权限越权、附件泄露、生产站不可用。
- P1：好友申请、工作区成员管理、登录设备、附件、通知或搜索核心路径不可用。
- P2：体验不顺、文案不清楚、移动端布局拥挤。
- P3：后续优化或非阻塞功能建议。

## v0.4 稳定化方向

- 基础错误记录：登录失败、消息发送失败、附件上传失败、通知失败、成员管理失败。
- 轻量管理员操作记录：添加成员、移除成员、角色变更、退出其他设备。
- onboarding 优化：管理员添加未注册邮箱时明确提示“对方需先注册”。
- 生产维护材料：备份/恢复检查、测试数据清理 SOP、试用问题日报模板。

## 暂不进入 v0.3.1

- 视频动态头像。
- 语音/视频通话。
- 频道和机器人不进入当前产品路线图。
- 端到端加密。
- 企业 SSO、合规归档、审计报表。
