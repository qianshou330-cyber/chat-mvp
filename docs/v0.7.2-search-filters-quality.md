# v0.7.2-beta Search Filters and Result Quality

`v0.7.2-beta` adds lightweight filters on top of the v0.7 server-backed search layer. It keeps the current Supabase RPC approach and does not introduce channels, bots, voice/video calls, end-to-end encryption, or a dedicated enterprise search service.

## What Changed

- `search_messages_v2` adds optional filters for message type, sender, and created-at range.
- Global search exposes message type and time filters.
- Current-conversation search exposes message type, sender, and time filters.
- Search runs only for meaningful queries: Chinese can search from one character; English/numeric queries need at least two characters.
- Results continue to show message type, sender, time, conversation, keyword highlight, and search-result navigation.

## Security Boundary

- `search_messages_v2` uses `can_access_conversation(..., auth.uid())`.
- Non-members cannot search private chats, group messages, attachments, group files, or management-only data.
- Deleted messages and hidden attachments are excluded from search results.
- Search results do not include signed URLs, tokens, secrets, or attachment file contents.

## Test Checklist

- Chinese keyword search returns saved historical messages.
- Type filters return only text/image/video/file results.
- Current-conversation sender filter returns only messages from the chosen member.
- Time filters narrow results to today, 7 days, or 30 days.
- Clicking a filtered result still opens the conversation and highlights the target message.
- Temporary smoke data uses `codex.v072.*@example.com` and `codex-v072-*`.
