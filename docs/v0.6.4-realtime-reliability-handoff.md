# v0.6.4 Realtime Reliability Handoff

## Summary

`v0.6.4-beta` is a reliability patch for company trial use. It improves weak-network behavior, Realtime recovery visibility, and failed-message handling without changing the database model.

## What Changed

- Chat screens now expose a lightweight connection state: `连接中`, `已连接`, `正在重连`, or `离线`.
- Browser offline state disables the composer and keeps the warning inside the current chat page.
- Supabase Realtime channel status updates refresh the app state after successful subscription and mark reconnecting states when a channel errors, times out, or closes unexpectedly.
- Text messages now keep a local failed bubble with `重试 / 移除` instead of disappearing on insert failure.
- Rapid duplicate text sends are guarded while a matching message is already pending.

## What Did Not Change

- No Supabase tables, RPCs, RLS policies, Edge Functions, or environment variables changed.
- No offline queue or background sync was added.
- File/video retry still depends on the current browser session retaining the selected file object.

## Smoke Focus

- A/B realtime send and receive across text, image, file, and video messages.
- Offline composer disabled state and online recovery refresh.
- Realtime reconnect refresh for current conversation and conversation list.
- Existing permissions: non-members cannot read private or group data.
