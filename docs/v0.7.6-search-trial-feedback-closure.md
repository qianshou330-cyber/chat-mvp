# v0.7.6-beta Search Trial Feedback Closure

`v0.7.6-beta` is a final search-experience closure patch for the v0.7 search series. It keeps the Supabase RPC search approach and does not add channels, bots, voice/video calls, end-to-end encryption, or a dedicated search service.

## Feedback Status

- GitHub Issues should be checked for search-related feedback before tagging this version.
- If no real user search feedback exists, record `暂无真实用户搜索反馈` and rely only on automatic production smoke results.
- P0/P1 search issues block the tag and require a minimal fix plus a repeated smoke test.

## UX Changes

- Search result empty states now suggest clearing filters, shortening the keyword, or switching scope.
- Global search shows `已加载全部匹配结果` when no more pages remain.
- Current-conversation search shows `已加载当前会话全部匹配结果` when no more pages remain.
- Returning from a search result keeps the global search keyword, filters, and result list in place.
- Search notices remain scoped to the search panel instead of the chat list home screen.

## Validation

- Run local checks: `npm run lint`, `npm run test -- --run`, `npm run build`, `npm run e2e`.
- Production smoke should cover Chinese keyword search, filters, result navigation, state retention after returning, load-more completion, and A/B/C permission regression.
- Temporary smoke accounts should use `codex.v076.*@example.com`; test uploads should use `codex-v076-*`.

## Boundaries

- Do not store raw search keywords, message bodies, attachment names, tokens, or secrets in feedback or error records.
- Keep v0.7.6 as a search UX and feedback closure patch unless production smoke exposes a P0/P1 requiring a minimal fix.
