# v0.3 公司试用版实现拆分

## 目标

把 `v0.2.1-beta` 从外测版推进到 20-30 人公司内部试用版。v0.3 只补公司试用基础能力，不做 Telegram 级频道、机器人、语音/视频、端到端加密或视频动态头像。

当前状态：v0.3 工作区、管理员成员管理和登录设备管理已完成生产 smoke test，准备冻结为 `v0.3-beta`。

跟踪 Issue：https://github.com/qianshou330-cyber/chat-mvp/issues/6

## 里程碑 1：团队/工作区

- 状态：已完成并通过生产 smoke test，新增 migration `20260618000000_workspaces.sql`。
- 新增工作区概念，让测试公司成员归属于同一个空间。
- 现有个人数据保留，首次进入时为已有用户创建默认工作区。
- 新建群聊在 v0.3 默认归属当前工作区；旧单聊和好友申请先保持个人社交模型，不重构成企业通讯录。
- 非工作区成员不能读取该工作区会话、成员列表或附件。

验收：A/B 同空间可协作；C 不在空间内时看不到 A/B 工作区数据。

## 里程碑 2：管理员成员管理

- 状态：已完成并通过生产 smoke test，第一版已接入个人资料页的“工作区管理”区域。
- 工作区角色默认分为 `owner`、`admin`、`member`。
- `owner/admin` 可通过已注册邮箱添加成员、查看成员、移除成员、调整 `admin/member` 角色。
- `member` 不能管理其他成员。
- 被移除成员会被同步移出该工作区群聊的 `conversation_members`，刷新后不能继续读取工作区群聊消息或附件。

验收：管理员移除成员后，该成员无法进入受限会话；普通成员不能移除别人。

## 里程碑 3：同账号多端登录管理

- 状态：已完成并通过生产 smoke test，第一版已接入个人资料页的“登录设备”区域，新增 migration `20260618010000_device_sessions.sql`。
- 保留 Supabase 多 session 能力。
- 新增设备记录：设备名称、浏览器、最后活跃时间、当前设备标识。
- 用户可查看自己的设备列表。
- 用户可退出其他设备。
- 用户可移除单台设备；第一版为 App 级撤销，该设备下次刷新或心跳时退出。
- 在线状态按设备聚合，区分在线、最近活跃和离线。

验收：同账号两台设备登录后都显示在设备列表；退出其他设备后，被退出设备需要重新登录。

## 里程碑 4：运营与外测管理

- 状态：基础收口已完成，生产 smoke test 和测试数据清理步骤见 `docs/v0.3-production-smoke-test.md`。
- 增加 App 内反馈入口，指向 GitHub Issues。
- 补生产 smoke test 清单和测试数据清理步骤。
- 记录关键错误：登录失败、消息发送失败、附件上传失败、通知订阅失败。
- 继续保持 P0/P1 分诊流程。

验收：测试者能从 App 找到反馈入口；管理员可以按文档清理测试数据。

## 里程碑 5：公司试用安全材料

- 状态：基础收口已完成，安全说明见 `docs/company-trial-safety.md`。
- 补隐私说明。
- 补数据保留说明。
- 补敏感信息提示。
- 补管理员使用说明。

验收：邀请 20-30 人试用前，测试者能看到当前能力边界和不支持项。

## 暂不进入 v0.3

- 视频动态头像：排到 `v0.5+`。
- 语音/视频通话、频道、机器人、大群优化、端到端加密。
- 企业级审计、SSO、合规归档和数据驻留。
