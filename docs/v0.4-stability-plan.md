# v0.4 稳定化计划

## 当前目标

`v0.4` 面向 20-30 人公司内部试用后的稳定化，不做视频动态头像、语音/视频或端到端加密；频道和机器人不进入当前产品路线图。重点是让管理员能看到关键错误和最近管理操作，便于试用期间定位问题。

## 已实现能力

- 新增 `app_error_events`：记录用户可见的关键错误。
- 新增 `admin_activity_logs`：记录管理员成员操作和退出其他设备操作。
- 个人资料页新增“管理员记录”，仅 owner/admin 可见。
- 管理员添加未注册邮箱时，提示“对方需先注册”。
- 日志上下文会过滤 `password`、`token`、`secret`、`body`、`filename` 等敏感字段。

## Supabase 接入

运行 migration：

```text
supabase/migrations/20260618020000_operational_logs.sql
```

验收：

- `app_error_events` 和 `admin_activity_logs` 存在且 RLS 开启。
- owner/admin 可读取当前工作区日志。
- member 不能读取管理员记录。
- 非工作区成员不能读取任何日志。

## 试用期使用方式

- P0/P1 仍进入 GitHub Issues。
- 管理员记录只用于定位问题，不替代正式审计。
- 不在日志中记录消息正文、附件文件名、密码、token、service role、VAPID private key 或 webhook secret。
- 如果日志 migration 尚未运行，App 会继续正常使用，只是管理员记录为空。
