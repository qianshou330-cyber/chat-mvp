# v0.7.4-beta Search Performance and Observability

`v0.7.4-beta` is a performance and observability patch for the v0.7 search layer. It keeps the `search_messages_v3` interface unchanged and does not introduce a dedicated search service.

## Changes

- Supabase migration: `20260620010000_message_search_performance.sql`.
- Adds `pg_trgm` support for contains-style text search.
- Adds trigram indexes for message body, conversation title, and profile display name.
- Adds filtered cursor indexes for message type, sender, conversation, and visible attachment lookup.
- Search RPC failures are recorded through the existing app error event flow with sanitized context only: scope, filter presence, pagination flag, conversation id when applicable, and query length.

## Validation

- Run local checks: `npm run lint`, `npm run test -- --run`, `npm run build`, `npm run e2e`.
- In Supabase, run `20260620010000_message_search_performance.sql`.
- Confirm `search_messages_v3` still returns the same columns as v0.7.3.
- Production smoke should cover Chinese keyword search, filters, load more, retry, and A/B/C permission regression.
- Temporary smoke accounts should use `codex.v074.*@example.com`; test uploads should use `codex-v074-*`.

## Boundaries

- Do not store raw search keywords in `app_error_events`.
- Do not add Meilisearch, Typesense, Elasticsearch, channels, bots, voice/video calls, or end-to-end encryption.
- This patch should not change RLS behavior or message result visibility.
