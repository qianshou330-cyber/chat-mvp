# v0.7.5-beta Search UX and Trial Feedback

`v0.7.5-beta` is a search UX and feedback convergence patch after the v0.7.3/v0.7.4 search pagination and performance work. It does not add channels, bots, voice/video calls, end-to-end encryption, or a dedicated enterprise search service.

## Feedback Status

- GitHub Issues search for `search OR 搜索` found only historical planning / external-test tracking issues, not a current v0.7 search-specific P0/P1.
- There is no confirmed real-user search regression feedback at the time of this handoff.
- This patch is therefore based on automatic smoke findings and UX review only; it must not be presented as real user feedback.

## UX Changes

- Global search results show the scope as `全部会话` and the currently displayed result count.
- Current-conversation search results show the scope as `当前会话` and the currently displayed result count.
- Filter bars show a short selected-filter summary such as type, sender, or date range.
- Load-more areas explain that more results are available and suggest narrowing filters when result sets are large.
- Search errors remain scoped to the search panel and keep retry actions local.

## Validation

- Run local checks: `npm run lint`, `npm run test -- --run`, `npm run build`, `npm run e2e`.
- Production smoke should cover Chinese keyword search, type/sender/date filters, load more, result navigation, and A/B/C permission regression.
- Temporary smoke accounts should use `codex.v075.*@example.com`; test uploads should use `codex-v075-*`.

## Boundaries

- Do not store raw search keywords, message bodies, attachment names, tokens, or secrets in feedback or error records.
- Keep v0.7.5 as a UI and feedback convergence patch unless production smoke exposes a P0/P1 requiring a minimal fix.
