# v0.8.0-beta Production Operations Handoff

`v0.8.0-beta` is a production-hardening patch after `v0.7.7-beta`. It does not add chat features, database schema, Supabase RPCs, RLS policies, Edge Functions, or environment variables. It exists to make the current production baseline easier to operate, verify, recover, and hand off without real company-trial feedback.

## Baseline

- Source baseline: `v0.7.7-beta`.
- Production URL: `https://chat-mvp-tau.vercel.app`.
- Temporary smoke accounts: `codex.v080.*@example.com`.
- Temporary smoke uploads: `codex-v080-*`.
- Real trial feedback status: skipped for this release; do not present automatic smoke as real user feedback.

## Daily Production Checklist

- Auth: sign-up/sign-in works, no spike in visible login errors.
- Messages: direct and group messages send, Realtime updates arrive, failed messages can retry or be removed.
- Media: image, file, and video uploads complete; hidden/deleted attachments do not produce previews or signed URLs.
- Search: current/all conversation search works with filters, load more, and result navigation.
- Notifications: at least one real Chrome subscription can receive a generic notification; payload omits message body, attachment name, and email.
- Permissions: owner/admin/member/non-member boundaries still hold for messages, attachments, group files, push subscriptions, and management records.
- Operations: group management records, trial inspection counts, and app error events remain readable only to allowed admins.

## Recovery Drill Checklist

- Confirm Supabase migrations are applied in filename order and all user-data tables still have RLS enabled.
- Confirm Storage buckets exist with expected access boundaries: `chat-uploads` private, `profile-avatars` public, `profile-avatar-videos` public.
- Confirm Edge Function `send-message-push` exists and its secrets are present only in Supabase Function secrets.
- Confirm Database Webhook or reviewed `pg_net` fallback calls the push function with `x-webhook-secret`.
- Confirm Vercel has only public frontend variables and no service-role key, VAPID private key, webhook secret, or database password.
- After any restore, run the production smoke before inviting users back in.

## Failure Handling

- P0/P1 issues block `v0.8.0-beta` and require the smallest safe fix, local checks, CI, Vercel deploy, and repeated Chrome production smoke.
- P2/P3 issues can be recorded for later unless they repeatedly block daily company use.
- Do not store raw search keywords, message bodies, attachment names, passwords, tokens, webhook secrets, VAPID private keys, service-role keys, or database passwords in this document.
