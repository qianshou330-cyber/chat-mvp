# 2026 年 7 月公司内部试用计划

## 目标

以 `v0.6.8-beta` 为基线，把当前聊天 MVP 推进到 20-30 人公司内部试用稳定版。7 月不扩张 Telegram 大功能，重点是让现有登录、群详情成员管理、聊天、图片/视频/文件、搜索、通知、登录设备、群管理记录和试用巡检在真实团队里稳定运行。

## 第 1 周：5 人先导试用

- 创建 1 个正式试用群聊，由 owner/admin 在群详情页添加 5 个核心用户。
- 每天使用 `docs/trial-daily-report-template.md` 记录核心路径状态。
- 启动前按 `docs/5-person-pilot-kickoff.md` 完成管理员、试用用户和 Day 1 检查准备。
- 所有反馈进入 GitHub Issues，并按 P0/P1/P2/P3 分类。
- P0/P1 只修稳定性、权限、通知、附件和成员管理问题，不夹带新功能。

先导必测路径：

- 登录、退出、重新登录。
- 群成员添加、移除、角色调整和邀请说明复制。
- 好友申请、同意、单聊。
- 群聊创建、群消息、成员移除后权限回归。
- 图片、视频和文件上传、打开和非成员访问限制。
- 搜索联系人、群名和中文消息关键词。
- Web Push 通知开启、接收和关闭。
- 登录设备列表、退出其他设备。
- owner/admin 查看群管理记录和试用巡检，member 不可见。

## 第 2 周：扩到 20-30 人

- 先导 1-3 天无 P0/P1 后扩容。
- 管理员通过已注册邮箱添加成员，不使用 `codex.*` 测试账号。
- 每日检查 Supabase Auth、Realtime、Storage、Edge Function/Webhook 和 Vercel 部署状态。
- 固定巡检生产 smoke、关键错误记录、群管理记录、试用巡检、Storage 使用量和通知失败情况。

扩容闸门：

- 连续 1-3 天无 P0。
- P1 均有复现步骤、处理决策和负责人。
- 管理员理解成员移除后的权限影响。
- 试用者已看到安全说明，知道当前不支持端到端加密、语音/视频和原生 App。

## 第 3 周：v0.6.x 稳定化

- 根据试用反馈发布 `v0.6.1-beta` 或 `v0.6.2-beta` 小修版本。
- 优先修复登录失败、消息丢失、附件/视频打不开、通知不触发、成员移除后仍可读、移动端阻塞。
- 优化管理员体验：未注册成员提示、邀请说明、试用巡检、错误提示、日报模板、测试数据清理 SOP。
- 每个修复必须经过本地检查、GitHub CI、Vercel Production smoke。

修复流程：

```text
复现 -> 最小修复 -> npm run lint -> npm run test -- --run -> npm run build -> push main -> 生产 smoke -> tag v0.6.x-beta
```

## 第 4 周：试用总结与 v0.6.x 决策

- 输出 7 月试用总结：活跃人数、P0/P1 数量、核心路径通过率、主要痛点。
- 若连续 7 天无 P0 且 P1 可控，冻结 `v0.6.x` 公司试用稳定基线。
- 只规划稳定性、性能、权限和运维小修，不直接开发大功能。
- 若媒体性能成为瓶颈，再单独规划缩略图/视频加载优化。

## 不进入 7 月范围

- 频道和机器人不进入当前产品路线图。
- 语音/视频通话、端到端加密、原生 App 暂不做。
- 企业级全文搜索服务、大群优化、企业级审计、SSO 和合规归档暂不做。
- 视频动态头像已作为 MVP 能力存在；7 月只做稳定性和性能验证，不扩张成语音/视频通话。

## 验收标准

- 5 人先导无 P0/P1 后扩到 20-30 人。
- 20-30 人试用连续 7 天无 P0。
- 所有 P1 均有复现步骤、处理决策和负责人。
- 非成员不能读取私聊、工作区群聊、附件、push subscription 或管理员记录。
- `npm run lint`、`npm run test -- --run`、`npm run build` 在每次修复后通过。
- 测试数据按 `docs/test-data-cleanup-sop.md` 清理，不遗留 `codex.*` 测试账号或测试附件。
