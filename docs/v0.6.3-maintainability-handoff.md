# v0.6.3 Maintainability Handoff

## Summary

`v0.6.3-beta` is an engineering stability patch. It keeps the product surface unchanged while reducing the risk around the large chat state hook and long message lists.

## What Changed

- `useChatApp()` still returns the same public shape used by `App` and UI components.
- Pure helpers were moved into `src/hooks/chatApp/`:
  - device/session metadata helpers
  - friendly error and missing-schema detection
  - sanitized logging helpers
  - Supabase row mappers
  - state upsert, permission, and message mutation helpers
  - global chat search helpers
- The chat view now renders a recent-message window first:
  - default render count: 80 latest messages
  - each “加载更早消息” action loads 80 more
  - current-conversation search only searches the loaded window and shows a scope note when older messages are hidden

## What Did Not Change

- No Supabase tables, RPCs, RLS policies, Edge Functions, or environment variables changed.
- No service-side pagination was added.
- No virtual list dependency was added.
- Realtime, Web Push, media uploads, group permissions, and device sessions keep the existing data flow.

## Follow-Up

- v0.7 adds service-side message pagination and server-side search; if real company-trial data outgrows contains matching, evaluate a dedicated enterprise search service later.
- Keep `useChatApp()` interface stable until production smoke confirms the extraction has no data-flow regressions.
