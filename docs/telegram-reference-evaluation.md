# DrKLO/Telegram 参考评估

## 结论

`DrKLO/Telegram` 可以作为成熟聊天产品的交互参考库，但不作为当前 PWA 项目的代码依赖、素材来源或协议来源。当前项目继续使用 React、Vite、Supabase 和 Vercel，用 clean-room 方式重写需要的体验。

参考来源：

- GitHub: <https://github.com/DrKLO/Telegram>
- License: <https://github.com/DrKLO/Telegram/blob/master/LICENSE>
- Figma Community reference: Telegram UI Screens Community, used only for visible structure, spacing, and information density.

## 不能直接使用

- 不复制 Java、Kotlin、C、C++、Go 或 Gradle 代码。
- 不复制 Telegram 名称、Logo、纸飞机图标、专有图标、贴纸、动画或资源文件。
- 不复制翻译文本、字符串资源或界面文案。
- 不引入 MTProto、Telegram API 或 Telegram 账号体系。
- 不把 GPL-2.0 源码混入当前 PWA 代码库。
- 不复制 Figma Community 文件中的 Telegram 品牌素材、图标、截图、组件实例或可识别视觉资产。

原因：

- 该仓库是 Android 原生项目，技术栈与当前 PWA 不匹配。
- 仓库标注 GPL-2.0，直接复用代码会带来强许可证义务。
- Telegram README 明确要求不要使用 Telegram 名称或标准 Logo，并提醒发布代码以符合许可证要求。

## 可以借鉴的方向

以下内容只记录交互结论，不复制实现代码或素材。

### 聊天列表

- 会话行密度：头像、名称、最后消息、时间、未读数。
- 最后消息类型前缀：图片、视频、文件、已删除消息。
- 右侧时间和未读数固定对齐。
- 移动端首屏显示更多会话，避免大卡片化。

### 聊天页

- 顶部栏结构：返回、头像、标题、在线状态、搜索或资料入口。
- 消息气泡节奏：左右气泡宽度、圆角、时间位置、发送状态。
- 媒体消息：图片缩略图、视频卡片、文件卡片。
- 上传状态：进度条、失败提示、重试入口。
- 图片查看器：点击打开、点击背景关闭、保存图片、打开原图。

### 群详情页

- 群信息顶部：头像、群名、成员数、编辑入口。
- 成员列表：添加成员、角色、在线状态、管理操作。
- 更多菜单：禁言、清理、举报或退出等可扩展操作。
- 群文件：媒体、文件、链接等分组的轻量入口。
- 群公告和置顶内容：放在群信息和消息流的可见位置。

### 群权限模型

- 角色边界：owner、admin、member。
- 全体禁言：owner/admin 可发言，member 不能发言。
- 单人禁言：只限制指定成员，不影响其他成员。
- 消息删除：普通成员撤回自己消息，管理员治理群消息。
- 群文件治理：隐藏附件、保留审计记录、不暴露 signed URL。

### 媒体体验

- 图片预览和保存。
- 视频消息卡片。
- 头像和媒体上传进度。
- 上传失败后的本地状态清理。
- 缩略图优先，原图/原视频按需打开。

## 当前项目映射

### v0.5.3-beta：群禁言

- 全体禁言。
- 单个成员禁言。
- 输入区禁用和中文原因提示。
- Supabase RLS 发送兜底。
- 管理员记录：禁言和解除禁言。

### v0.5.4-beta：群成员与权限细节

- 群详情页内添加/移除成员。
- 群角色调整。
- 管理按钮只对 owner/admin 显示。
- member 不能通过 UI 或 RPC 越权。

### v0.5.5-beta：媒体查看器与上传细节

- 图片全屏预览。
- 长按保存图片。
- 上传进度和失败重试。
- 视频消息卡片体验优化。

### v0.5.6-beta：媒体体验与个人资料页 UI 收口

- 合并 v0.5.5 媒体体验改动，不单独拆 tag。
- 个人资料页改成轻量设置页：头像操作菜单、设置列表、邮箱只读、登录设备折叠。
- 图片预览、保存/打开原图、上传进度、失败重试和视频消息卡片完成 smoke。
- 媒体错误提示只出现在当前聊天或个人资料页，不回到聊天首页。

### v0.5.7-beta：群治理体验验收

- 群公告、置顶消息、删除消息、群文件和隐藏附件完成生产 smoke。
- 群禁言、单人禁言和群管理记录继续保持 owner/admin/member 权限边界。
- 群详情页以分组信息架构承载治理能力，不重新暴露独立工作区后台。
- 被删除或隐藏的附件不显示预览、不生成 signed URL。

### v0.5.8-beta：详情页视觉重做

- 群详情页和个人资料页采用 Telegram 式轻顶部、头像摘要、紧凑操作条和分组列表节奏。
- 管理操作降噪为行内操作或“更多管理”，普通成员只看到只读详情。
- 个人资料页保持设置页结构，头像操作使用轻量菜单，不回到后台表单样式。
- 继续只借鉴结构、密度和交互节奏，不复制 Telegram 或 Figma 素材。

### v0.5.9-beta：聊天主界面视觉收口

- 聊天列表采用紧凑会话行：头像、标题、最后消息、时间和未读数稳定对齐。
- 聊天页采用轻顶部栏、浅蓝/白色气泡、轻量系统提示和紧凑底部输入区。
- 图片、视频和文件消息保持现有业务逻辑，仅统一媒体卡片和气泡节奏。
- 当前会话搜索和空状态保持轻量，不新增服务端全文搜索。

### v0.6.0-beta：公司试用稳定版

- 不继续追 Telegram 新功能，先稳定 20-30 人公司试用的核心路径。
- 群详情加入可复制邀请说明，帮助管理员引导未注册成员先注册再入群。
- 群详情“更多管理”加入试用巡检摘要，观察最近错误、通知失败、附件失败和管理操作。
- 媒体和头像继续使用懒加载、进度和失败重试策略，降低多群和多媒体消息下的移动端压力。

### v0.6.1-beta：公司试用收口补丁

- 不新增 Telegram 式功能，只清理 v0.6 自动 smoke 数据、补真实 Chrome Web Push smoke、同步公司试用文档。
- 记录生产 smoke 结果，作为进入真实 20-30 人试用前的交接凭据。
- 大 bundle 拆分、服务端搜索或更重的性能工程另行进入 v0.6.2+，不夹在收口补丁里。

### v0.6.2-beta：性能与可维护性收口

- 不新增 Telegram 式功能，只拆分群详情页、个人资料页和通用展示工具，降低首屏主 bundle。
- 保持 Telegram/Figma clean-room 边界不变：只参考结构节奏，不复制源码、Logo、图标、资源或文案。
- 将 PWA service worker 切到 `iife` 构建，消除 `inlineDynamicImports` 警告，并继续用生产 smoke 覆盖 Web Push。

## 开发守则

- 只从公开界面和产品行为总结需求，不阅读或移植具体源码实现。
- 每项功能都用当前项目自己的 React 组件、Supabase RPC、RLS 和 Storage policy 实现。
- 所有用户可见文案默认简体中文。
- 不新增频道和机器人，它们不进入当前产品路线图。
- 不做语音/视频通话或端到端加密，除非后续单独重新立项。
- 如果未来确实要使用任何第三方素材，必须先确认许可证、署名要求和是否适合商业/闭源分发。

## 验收清单

- 代码库中不出现 Telegram 源码、品牌素材、Logo、专有图标或资源文件。
- 新功能测试覆盖权限边界和非成员访问。
- 每次涉及群权限或媒体治理的版本都跑：

```bash
npm run lint
npm run test -- --run
npm run build
```

- 生产 smoke 覆盖：登录、好友申请、单聊、群聊、附件、搜索、通知、登录设备、管理员记录、群详情管理。
