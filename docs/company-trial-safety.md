# 公司试用安全说明

## 适用范围

本文档面向 `v0.8.1-beta` 及后续公开公测。当前版本适合验证轻量聊天、群管理、附件/媒体、搜索、通知、登录设备、运行状态和管理员记录，不等同于企业级合规通讯产品。

## 隐私说明

- 消息、会话、成员关系、附件元数据和设备记录存储在 Supabase。
- 附件存放在 Supabase Storage，下载通过成员权限和 signed URL 控制。
- Web Push 通知默认只显示“你有一条新消息”，payload 不包含消息正文、附件名或邮箱。
- 视频动态头像只用于头像展示，不提供语音或视频通话能力；聊天列表默认显示静态封面以控制性能。
- 当前版本不支持端到端加密，项目维护者和云服务管理员在必要排障时可能具备基础设施层面的访问能力。

## 数据保留说明

- 当前版本没有自动数据保留周期，也没有自动过期删除策略。
- 测试数据由管理员按 `docs/test-data-cleanup-sop.md` 中的清理步骤手动删除。
- 删除 Auth 用户后，部分业务数据会按数据库外键策略级联清理；清理生产数据前必须先确认影响范围。
- 附件文件需要在 Storage 中单独确认和清理，避免遗留测试文件。

## 敏感信息提示

试用期间不要发送以下内容：

- 密码、API key、service role key、数据库密码、VAPID private key 或 webhook secret。
- 身份证、护照、银行卡、合同原件、医疗记录、法律争议材料等高度敏感信息。
- 公司未公开财务、客户名单、源代码凭据或其他不适合放入 beta 工具的信息。

## 管理员使用说明

- 角色分为 `owner`、`admin`、`member`。
- `owner/admin` 可以在群详情页通过已注册邮箱添加群成员、移除成员、调整 `admin/member` 角色、禁言普通成员并查看管理记录。
- `member` 只能查看群信息、成员和有权限的文件，不能管理成员或读取管理记录。
- 移除群成员时，系统会同步移除该成员在对应群聊中的成员关系。
- 被移除成员刷新后不能继续读取该群消息和附件。
- 旧好友申请和单聊仍是个人社交模型，暂不等同于企业通讯录。

## 登录设备说明

- 用户可在个人资料页查看自己的登录设备。
- “退出其他设备”会保留当前设备，并尝试退出同账号其他设备。
- “移除”单台设备是 App 级撤销，被移除设备下次刷新或心跳后退出。
- 设备管理第一版用于公司内部试用，不提供企业级审计、地理位置记录或强合规报告。

## 当前不支持

- 端到端加密。
- 原生 iOS/Android App。
- 语音通话、视频通话。
- 频道和机器人不进入当前产品路线图。
- 企业 SSO、合规归档、数据驻留、审计报表。
