# v0.4.2-beta 稳定性补丁

## 目标

`v0.4.2-beta` 面向 5 人先导试用后的第一轮小修，只处理稳定性、权限、通知、附件、成员管理和移动端阻塞问题，不扩张新功能。

## 本轮补丁范围

- 消息或附件发送失败后，清理本地“发送中”状态，避免测试者误以为仍在排队。
- 通知开启/关闭失败时，显示中文可理解原因，并继续写入脱敏错误记录。
- 工作区成员移除或角色更新成功后，立即刷新本地列表和角色状态。
- 工作区成员管理操作中禁用重复点击，降低重复提交风险。
- 添加成员表单明确提示：对方需要先完成注册，才能加入工作区。
- 管理员记录和关键错误记录增加更明确的空状态说明。

## 不做

- 不做视频动态头像。
- 不做频道和机器人。
- 不做语音/视频通话。
- 不做端到端加密。
- 不改核心数据模型，除非后续发现权限或数据一致性 P0/P1。

## 验收清单

- 本地通过 `npm run lint`、`npm run test -- --run`、`npm run build`。
- 生产 smoke 覆盖登录、工作区、成员管理、好友申请、单聊、群聊、附件、搜索、通知、登录设备和管理员记录。
- 非成员不能读取私聊、工作区群聊、附件、push subscription 或管理员记录。
- 被移除成员刷新后不能继续访问工作区群聊和附件。
