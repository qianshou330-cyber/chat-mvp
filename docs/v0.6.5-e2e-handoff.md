# v0.6.5-beta E2E Handoff

## Summary

`v0.6.5-beta` adds repeatable Playwright smoke coverage before company trial expansion. It does not change product behavior, database schema, Supabase policies, Edge Functions, or Vercel environment variables.

## Local Commands

```bash
npm run lint
npm run test -- --run
npm run build
npm run e2e
npm run e2e:headed
```

`npm run e2e` starts a local Vite server through `playwright.config.ts` and runs against Demo mode. No real production account, password, Supabase service role, VAPID private key, or webhook secret is stored in the test suite.

The tracked `.env.e2e` file only sets `VITE_FORCE_DEMO_MODE=true` so local and CI E2E do not accidentally use a developer machine's `.env.local` Supabase project.

## Covered Paths

- Demo sign-in and chat list load.
- Direct chat send flow and current-conversation search.
- Group details entry, group permissions, group files, trial inspection, and group management records.
- Image message preview, image viewer actions, and video message card rendering.
- Browser offline state, disabled composer, and online recovery.

## Not Covered by Local E2E

- Real Supabase A/B/C account permissions.
- Real Realtime delivery across separate authenticated users.
- Real Web Push delivery.
- Supabase Storage signed URL permission checks.

Those remain production Chrome smoke checks because they require real cloud state and must not put reusable credentials into CI.

## Temporary Data

If a production smoke for this version needs temporary users, use:

- `codex.v065.*@example.com`
- `codex-v065-*` upload prefixes

Clean them with `docs/test-data-cleanup-sop.md` after smoke.
