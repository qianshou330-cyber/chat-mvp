# v0.3 公司试用版计划

## 目标

把当前 `v0.2-beta` 从小范围外测版推进到 20-30 人公司内部试用版。目标是支撑一个小团队完成日常轻量沟通、文件分享、通知提醒和基础成员管理，不追求 Telegram 级完整功能。

## 公司试用最低标准

- 生产站点核心路径连续 3 天无 P0。
- P1 问题都有负责人、复现步骤和修复计划。
- 至少 3 名真实外测用户跑通注册、好友申请、单聊、群聊、附件、搜索和通知。
- 管理员可以查看成员、移除成员，并理解移除后的权限影响。
- 有隐私说明、数据保留说明、敏感信息提示和管理员使用说明。

## v0.3 功能范围

- 团队/工作区：一个用户可以加入一个公司空间，空间内成员用于联系人和群聊管理。
- 成员管理：管理员可邀请、查看、移除成员；被移除成员不能继续读取新会话或新附件。
- 多端登录：保留 Supabase 多 session，新增设备列表、当前设备标识、退出其他设备和在线状态聚合。
- 运营能力：补生产 smoke test 清单、测试数据清理步骤、错误反馈入口和基础监控记录。
- 安全文档：明确当前不支持端到端加密、语音/视频、频道、机器人和原生 App。

工程拆分见 `docs/v0.3-implementation-backlog.md`。

## 暂不进入 v0.3

- 视频动态头像，排到 `v0.5+`。
- 语音/视频通话、频道、机器人、大群优化、端到端加密。
- 企业级审计、SSO、合规归档和数据驻留。

## 验收

- A/B/C 三账号权限回归：非成员不能读私聊、附件或 push subscription。
- 管理员移除成员后，被移除账号刷新页面不能继续访问受限会话。
- 同账号多端登录后，设备列表显示当前设备和其他设备，并可退出其他设备。
- 30 人以内测试空间能完成建群、发消息、上传附件和接收通知。
