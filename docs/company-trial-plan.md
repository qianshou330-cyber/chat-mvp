# v0.6 公司试用版计划

## 目标

以 `v0.6.0-beta` 为公司内部试用基线，支撑 20-30 人小团队完成日常轻量沟通、群管理、文件/媒体分享、通知提醒、成员邀请、登录设备管理和管理员试用巡检。不追求 Telegram 级完整功能。

## 公司试用最低标准

- 生产站点核心路径连续 3 天无 P0。
- P1 问题都有负责人、复现步骤和修复计划。
- 至少 3 名真实或自动模拟测试用户跑通注册、好友申请、单聊、群聊、图片/视频/文件、搜索、通知和登录设备管理。
- 管理员可以在群详情页添加/移除成员、调整角色、禁言成员、复制邀请说明、查看试用巡检，并理解移除后的权限影响。
- 有隐私说明、数据保留说明、敏感信息提示和管理员使用说明。

## v0.6 功能范围

- 群详情内管理：成员添加/移除、角色调整、群禁言、邀请说明、群管理记录和试用巡检。
- 聊天和媒体：单聊、群聊、图片预览、视频消息、文件上传、视频动态头像和上传失败重试。
- 通知和设备：Web Push 通知、登录设备列表、退出其他设备和在线状态聚合。
- 运营能力：生产 smoke test 清单、测试数据清理 SOP、关键错误记录、管理员操作记录和试用日报模板。
- 安全文档：明确当前不支持端到端加密、语音/视频、原生 App、频道和机器人。

生产 smoke 和测试数据清理见 `docs/deployment-checklist.md`、`docs/test-data-cleanup-sop.md` 和 `docs/v0.6.1-production-smoke.md`。公司试用安全材料见 `docs/company-trial-safety.md`。5 人先导执行见 `docs/5-person-pilot-kickoff.md`。

## 暂不进入 v0.6

- 语音/视频通话、大群优化、端到端加密；频道和机器人不进入产品路线图。
- 企业级审计、SSO、合规归档和数据驻留。

## 验收

- A/B/C 三账号权限回归：非成员不能读私聊、附件或 push subscription。
- 管理员移除成员后，被移除账号刷新页面不能继续访问受限会话。
- 同账号多端登录后，设备列表显示当前设备和其他设备，并可退出其他设备。
- 30 人以内测试空间能完成建群、发消息、上传图片/视频/文件、接收通知和查看试用巡检。
