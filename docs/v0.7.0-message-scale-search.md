# v0.7.0-beta Message Scale and Server Search

## Summary

`v0.7.0-beta` adds the first server-backed message history and search layer for company-trial scale. It does not add channels, bots, voice/video calls, end-to-end encryption, SSO, or compliance archiving.

## What Changed

- `get_conversation_messages(conversation_id, before_created_at, page_size)` returns one authorized message page with attachment metadata.
- `search_messages(search_query, conversation_id, result_limit)` searches authorized saved messages with Chinese-friendly contains matching.
- Supabase mode loads recent message pages per conversation instead of relying on a full message dump.
- Chat view loads earlier history from Supabase when available.
- Current-conversation search uses the server when configured; Demo mode keeps local loaded-message search.
- Global search merges local conversation matches with server message matches and deduplicates results.

## Security Boundary

- RPC functions call `can_access_conversation`, so non-members cannot page or search private chats, group messages, attachments, group files, or management-only data.
- Deleted messages and hidden/deleted attachments are excluded from server search.
- Attachment signed URLs still come from the existing client-side signed URL path and are not stored in search results.
- The migration adds no service role key, Edge Function, Vercel variable, or frontend secret.

## Deployment Notes

- Run `supabase/migrations/20260619000000_message_pagination_search.sql` before production smoke.
- Run `supabase/migrations/20260619010000_message_search_context.sql` after v0.7.0 to enable v0.7.1 search-result context loading.
- If the migration is missing, the client shows a Chinese migration warning and temporarily falls back to the old message query path.
- Temporary smoke users must use `codex.v070.*@example.com`; test uploads must use `codex-v070-*`.

## Test Checklist

- A/B can open a long direct chat and load earlier messages without duplicates.
- A/B can open a group with 100+ messages and load earlier messages.
- Current conversation search finds older saved Chinese messages even if they are not in the first rendered page.
- Global search finds message results across authorized conversations.
- C cannot page or search A/B private messages, group messages, attachments, group files, push subscriptions, or management records.
- Deleted messages and hidden/deleted attachments do not appear in search results.
- Existing flows still pass: login, friend request, direct chat, group chat, group mute, media, notifications, login devices, management records, trial inspection.

## Known Limits

- Search uses contains matching for Chinese reliability, not a dedicated search service.
- Server-side ranking is simple newest-first ordering.
- Message snippets are not highlighted by the server; highlighting remains a UI concern.
- If company-trial data grows beyond this approach, evaluate dedicated indexes or a search service in a later version.
