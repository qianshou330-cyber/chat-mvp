# v0.7.1-beta Search Context and History Reliability

`v0.7.1-beta` is a reliability patch for the v0.7 server-backed message history and search layer. It does not add channels, bots, voice/video calls, end-to-end encryption, SSO, or compliance archiving.

## What Changed

- Search results now carry `message_id` and `message_type`, so the UI can show a clearer type label and navigate to the matching message.
- `get_conversation_message_context(conversation_id, message_id, page_size)` loads an authorized page ending near the target message, which helps open older search results without blindly paging many times.
- Current-conversation search results can load context, close the search panel, scroll to the target message, and briefly highlight it.
- Older-message loading keeps the scroll anchor stable and de-duplicates messages when Realtime inserts and pagination overlap.

## Supabase Deployment

- Run `supabase/migrations/20260619000000_message_pagination_search.sql` first.
- Then run `supabase/migrations/20260619010000_message_search_context.sql`.
- Verify both RPCs exist:

```sql
select proname
from pg_proc
where proname in (
  'get_conversation_messages',
  'search_messages',
  'get_conversation_message_context'
);
```

## Security Boundary

- All RPCs call `can_access_conversation(..., auth.uid())`.
- Non-members cannot page, search, or load context for private or group conversations.
- Deleted messages and hidden attachments are excluded from search results.
- Search returns IDs and snippets only; it never returns signed URLs, secrets, tokens, or attachment file contents.

## Test Checklist

- Search a Chinese keyword in a long conversation and open the result; the matching message should become visible and highlighted.
- Click “加载更早消息” multiple times; messages should remain ordered and not duplicate.
- A/B can search and page their own conversations; C cannot search or page A/B private or group data.
- Hidden or deleted attachments do not appear as previewable search results.
- If a search result cannot be context-loaded, the chat page shows a local Chinese retry hint and sending still works.
