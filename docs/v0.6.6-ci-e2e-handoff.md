# v0.6.6-beta CI E2E Handoff

## Summary

`v0.6.6-beta` turns the v0.6.5 Playwright smoke into a CI gate. It is a testing and trial-governance patch only: no product feature, Supabase schema, RLS, Edge Function, or environment-variable change.

## CI Behavior

GitHub Actions now runs:

```bash
npm ci
npm run lint
npm run test
npm run build
npx playwright install --with-deps chromium
npm run e2e
```

If Playwright fails, CI uploads `playwright-report` and `test-results` as artifacts so the failing trace, screenshots, and HTML report can be inspected.

## Coverage Boundary

CI E2E runs local Demo mode only. It intentionally does not store production Supabase credentials, temporary production passwords, service-role keys, VAPID private keys, or webhook secrets.

Production smoke remains a separate Chrome validation step for:

- Real A/B/C account permissions.
- Realtime message delivery between different authenticated users.
- Web Push delivery.
- Storage signed URL permission checks.

## Production Smoke Naming

If production smoke needs temporary data, use:

- `codex.v066.*@example.com`
- `codex-v066-*` upload prefixes

Delete those accounts and uploads after validation using `docs/test-data-cleanup-sop.md`. Do not delete real trial users or business data.

## Release Gate

Create `v0.6.6-beta` only after:

- Local `lint/test/build/e2e` pass.
- GitHub Actions passes on `main`.
- Production smoke has no P0/P1.
- Temporary `codex.v065.*` or `codex.v066.*` data is either cleaned or explicitly recorded for cleanup.
