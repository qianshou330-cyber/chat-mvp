# v0.6.2-beta 性能与可维护性收口记录

## Summary

`v0.6.2-beta` 是公司试用前的性能和可维护性补丁，不新增业务功能、不新增 Supabase migration。重点处理 `v0.6.1-beta` 遗留的主 bundle 过大警告和 PWA service worker `inlineDynamicImports` 弃用警告。

## 代码收口

- 将群详情页、个人资料页和头像展示拆到独立组件文件，降低 `App.tsx` 的维护负担。
- 群详情页和个人资料页使用 `React.lazy` 懒加载，减少首屏主 bundle。
- 将 UI 格式化工具集中到 `src/lib/uiFormat.ts`，避免后续在页面组件里继续堆辅助函数。
- `useChatApp.ts` 暂不拆分，避免在公司试用基线前改动核心数据流。

## 构建结果

| 项目 | v0.6.1-beta | v0.6.2-beta |
| --- | --- | --- |
| 主 JS bundle | 约 `514.12 kB` | 约 `488.66 kB` |
| 群详情懒加载 chunk | 无 | 约 `14.97 kB` |
| 个人资料懒加载 chunk | 无 | 约 `11.39 kB` |
| PWA SW 格式 | `es` | `iife` |
| `inlineDynamicImports` 警告 | 存在 | 已消除 |

## 验收重点

- PWA `manifest.webmanifest` 和 `sw.js` 仍正常生成。
- Web Push 的 `push` 和 `notificationclick` 处理逻辑未改动。
- 登录、聊天列表、单聊、群聊、群详情、个人资料页仍可正常打开。
- 群禁言、群成员权限、媒体上传、通知、登录设备和管理员记录不回归。

## 后续建议

如果 20-30 人试用中继续发现首屏或聊天流卡顿，`v0.6.3+` 再考虑拆分 `useChatApp.ts` 数据层、对消息列表做虚拟滚动，或进一步拆分媒体处理代码。
