# Chat MVP v0.8.1 公开公测计划

## 发布快照

- 当前候选版本：`v0.8.1-beta`
- 基线来源：`v0.8.0-beta` 生产硬化版
- 计划上线日期：2026-07-01
- 生产地址：`https://chat-mvp-tau.vercel.app`
- 公测规模：约 30 人
- 注册策略：公开链接开放注册，不启用访问码或白名单

## 公测前提示

当前是公开公测版，不是正式企业合规通讯产品。测试者不要发送密码、密钥、客户资料、未公开财务信息、源码凭据或其他敏感内容。

当前已知限制：

- 暂不支持端到端加密。
- 暂不提供原生 iOS 或 Android App。
- 暂不支持语音通话、视频通话。
- 当前产品路线图不包含频道和机器人。
- Web Push 第一版重点验收 Chrome/Edge 桌面和 Android Chrome；iOS/Safari 需要单独兼容验收。
- 搜索使用 Supabase RPC 和包含匹配，不是企业级全文搜索服务。

## 测试者路径

请每位测试者在生产地址使用邮箱和密码创建账号，然后完成：

1. 登录、退出、重新登录。
2. 修改昵称、简介、图片头像或视频头像。
3. 发送好友申请、同意申请、进入单聊。
4. 创建或加入群聊，发送文字、图片、视频和文件。
5. 打开图片预览、视频卡片和文件下载。
6. 使用聊天列表搜索、当前会话搜索、类型/发送人/时间筛选。
7. 在支持浏览器里开启通知，确认收到“你有一条新消息”。
8. 查看登录设备。
9. 管理员验证添加成员、移除成员、群禁言、群管理记录和运行状态。

## 反馈格式

所有反馈进入 GitHub Issues，并按优先级标记：

- P0：生产不可用、无法登录、权限越权、附件泄露、消息大面积失败。
- P1：核心路径不可用，例如好友申请、群聊、通知、附件、搜索、登录设备、群管理失败。
- P2：明显体验问题，例如移动端拥挤、提示位置不对、文案不清楚、状态刷新弱。
- P3：后续建议或非阻塞优化。

每条反馈建议包含：

- 测试者代号或邮箱。
- 设备和浏览器。
- 期望发生什么。
- 实际发生什么。
- 截图或录屏。

## 上线闸门

上线前必须通过：

- `npm run lint`
- `npm run test -- --run`
- `npm run build`
- `npm run e2e`
- Chrome 生产 A/B/C smoke
- Web Push 真实通知 smoke
- `codex.v081.*` 临时账号和 `codex-v081-*` 测试附件清理路径

若出现 P0/P1，不冻结 `v0.8.1-beta`，先做最小修复并重新验收。
