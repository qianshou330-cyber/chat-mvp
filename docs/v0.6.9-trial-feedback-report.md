## Required Regression Checks

Run these after every v0.6.9 fix candidate:

- Login, logout, and re-login.
- Friend request, direct chat, and group chat.
- Group details: add member, role change, mute/unmute, management records, trial inspection.
- Media: image preview, video card, file attachment, upload progress, retry/remove failure state.
- Notifications: generic Web Push notification, no message body, filename, or email in payload.
- Device management: current device, other device listing, revoke behavior.
- Permission boundaries: member cannot see admin-only records; non-member cannot read private chats, group messages, attachments, group files, push subscriptions, or management records.

## Patch Rules

- P0/P1: pause expansion, create or link a GitHub Issue, make the smallest safe fix, rerun local checks, CI, Vercel deploy, and Chrome production smoke.
- P2: collect for batching unless it blocks repeated daily use or appears in multiple trial reports.
- P3: keep as backlog only.
- Do not add channels, bots, voice/video calls, end-to-end encryption, or unrelated product expansion.
- Do not store trial passwords, service-role keys, VAPID private keys, webhook secrets, or database passwords in docs, CI, Vercel frontend variables, or GitHub.
