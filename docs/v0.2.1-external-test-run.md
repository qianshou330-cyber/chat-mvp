### Chrome Web Push 验收

- 账号：自动账号 B。
- 结果：通过。
- 过程：B 在真实 Chrome 个人资料页关闭并重新开启通知，Supabase 生成 1 条 `enabled=true` 的 push subscription。
- 发送：A 向 A/B 单聊插入 `Chrome 通知验收 ...` 测试消息。
- 验证：B 的 `push_subscriptions.last_sent_at` 更新，`last_error` 为空；A 没有启用的发送者订阅。
- 备注：自动化无法直接截取操作系统通知弹窗，因此以浏览器订阅写入和 Supabase 推送发送成功作为本轮验收信号。
