## 生产 Smoke Test 清单

每次 v0.3 修复后，至少跑一遍：

- 注册或登录一个测试账号。
- 打开个人资料页，确认显示“工作区管理”和“登录设备”。
- 用同一账号在两个浏览器或无痕窗口登录，确认设备列表显示两台设备。
- 点击“退出其他设备”，确认当前设备保持登录，另一端刷新后退出。
- owner/admin 通过已注册邮箱添加成员。
- member 不能添加、移除或修改成员角色。
- owner/admin 移除成员后，被移除成员刷新后不能继续访问工作区群聊和附件。
- 回归好友申请、单聊、群聊、附件、搜索、通知和退出登录。
- 打开个人资料页，确认“试用说明”链接可访问公司试用安全说明。

## 测试数据清理

本轮自动测试账号使用前缀：

```text
codex.v03.device.
codex.v031.
```

清理建议：

- Supabase `Authentication -> Users` 中筛选 `codex.v03.device.` 和 `codex.v031.`，确认无人工测试仍依赖后再删除。
- Supabase `Storage -> chat-uploads` 中筛选测试文件名前缀 `device-smoke-`，确认后删除。
- Table Editor 中如需手动清理，先按测试账号 `user_id` 定位 `profiles`、`workspace_members`、`device_sessions`、`conversation_members`、`messages`、`attachments`。
- 不要执行没有前缀或没有 `user_id` 条件的批量删除。

## 已知限制

- 设备管理第一版是公司试用级，不是企业级设备审计。
- 单台设备移除是 App 级撤销，被移除设备在刷新或心跳后退出。
- `signOut({ scope: 'others' })` 会撤销其他设备的刷新会话，但已签发 access token 可能在短时间内继续有效，所以仍保留 `revoked_at` 守卫。
